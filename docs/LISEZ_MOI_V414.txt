MHUR France V414 — Correction des sessions Supabase

Corrections :
- Le jeton de connexion est renouvelé automatiquement avant son expiration.
- Une requête refusée avec « JWT expired » est relancée une fois après renouvellement.
- Si une ancienne session ne peut plus être renouvelée, les pages publiques (mods, builds, profils) continuent à charger en mode invité.
- Les actions privées demandent simplement de se reconnecter au lieu d’afficher le message technique Supabase.
- Le message trompeur demandant de relancer les SQL V387/V390 a été supprimé des erreurs temporaires.
- Le renouvellement est également vérifié quand l’utilisateur revient sur l’onglet ou retrouve Internet.

Aucune commande SQL supplémentaire n’est nécessaire.
